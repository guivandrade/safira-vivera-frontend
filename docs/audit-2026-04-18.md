## 1. O que já está bom ✅

- **Filtros globais** (período/plataforma/boost/inactive) aplicados uniformemente em 6+ lugares.
- **Preferências persistidas no backend** (`/settings/preferences`) com fallback localStorage.

- URL sync de filtros (bookmarkable, compartilhável)

- Humanização de erros do backend (Request failed 400 → "Meta rejeitou parte da consulta")
- Insights feed automático (detecta padrões acionáveis)

### Coerência de dados

## 2. Dívida técnica 🔧

### Crítica

- [ ] **Testes não rodam.** `vitest.config.ts` importa `@vitejs/plugin-react` mas o pacote **não está em `package.json`**. `npm run test` quebra. Job `test` no CI quebrado silenciosamente desde sempre. `test/setup.ts` referenciado mas não existe.
- [ ] **Zero unit tests.** Só um `test/e2e/home.spec.ts` placeholder. Com ~9k LOC e lógica crítica em `lib/insights.ts`, `lib/period.ts`, `lib/humanize-api-errors.ts`, `stores/*` — qualquer refactor é "deploy e reza".
- [ ] **Sem error boundary.** Crash em qualquer componente derruba a página inteira.
- [ ] **Sem monitoring.** Nenhum Sentry/Datadog/LogRocket. Vera pode ter erros que a gente nunca soube.

### Média

- [ ] **Código morto:** `GoalsCard.tsx`, `goals-store.ts`, `annotations-store.ts` existem mas não são renderizados em lugar nenhum.
- [ ] **Deps instaladas mas não usadas:** `react-hook-form`, `@hookform/resolvers`, `zod`.
- [ ] **`window.location.href`** em 2 lugares (DashboardOverview, CreativesPage) — força reload. Deveria ser `router.push`.
- [ ] **Keys de localStorage sem namespace consistente** — misturamos `safira-*`, `safira-cols-*`, `safira-filters`. Centralizar num helper.

### Pequena

- [ ] **`useAuth.ts`** provavelmente desatualizado/não usado — login page tem sua própria lógica.
- [ ] **Sem Storybook** — primitivos de UI sem documentação viva.
- [ ] **Type `Metrics` em `ComparePage`** duplicado do que já existe.

## 7. Visão 6 meses — "plataforma, não dashboard" 🚀

Sai de "painel de BI" e vira **centro de operação**. Vera faz ações no dashboard, não só olha.

- [ ] **Write-ops**: pausar/ativar campanhas, ajustar orçamento, duplicar criativo — tudo direto na UI via Meta/Google APIs. Exige auth escrita + confirmação robusta + audit log.
- [ ] **Budget optimizer**: sugestão automática "realocar R$500 de campanha X pra campanha Y vai render +18 conversões/semana baseado no histórico". Vera aceita com 1 clique.
- [ ] **Cross-platform optimizer**: quando Meta converte 2x mais barato que Google, sugere realocação.
- [ ] **White label / multi-tenant real**: agência cria contas pra N clientes. Template de dashboard por nicho (estética, odonto, advocacia).
- [ ] **Integração com CRM/agenda**: fechar o loop "conversão no ad → agendou no CRM → apareceu na consulta". Adicionar **LTV real**.
- [ ] **Público-alvo como feature**: permite Vera "clonar o público do criativo X que mais converte" direto do dashboard.
